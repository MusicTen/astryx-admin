# i18n 多语言支持设计(中/英)

日期:2026-07-10
状态:已确认

## 目标

为 astryx-admin 添加界面多语言支持,首期支持中文(zh)和英文(en)。当前全部 UI 文案为硬编码中文,约 187 处,分布在 26 个源文件(不含 mock 数据)。

## 决策记录

| 决策点 | 结论 |
| --- | --- |
| i18n 方案 | react-i18next + i18next |
| 默认语言 | 跟随浏览器(`navigator.language` 以 `zh` 开头 → 中文,其余 → 英文),用户手动切换后持久化 |
| 切换入口 | 顶栏快捷切换 + 设置→外观页,登录页也可切换 |
| mock 业务数据 | 不处理,i18n 只覆盖 UI 文案 |

## 架构

### 依赖

新增 `i18next`、`react-i18next` 两个运行时依赖(合计约 15kb gz)。不引入 language-detector 插件——检测逻辑只有几行,自己写。

### 文件结构

```
src/i18n/
├── index.ts          # i18next 初始化 + detectLanguage() 浏览器语言检测
└── locales/
    ├── zh.json       # 现有中文原文
    └── en.json       # 英文翻译
```

- 单一 namespace(`translation`),资源随主包直接打包,不做懒加载(文案量小)。
- 键按领域分层:`nav.*`、`common.*`、`login.*`、`dashboard.*`、`tasks.*`、`apps.*`、`users.*`、`settings.*`、`errors.*`。

### 语言状态

- `src/stores/ui.ts` 增加 `language: "zh" | "en"` 与 `setLanguage`,初始值取 `detectLanguage()`,随现有 `astryx-admin-ui` persist key 持久化到 localStorage。
- `AppProviders` 中用 effect 同步 `store.language` → `i18n.changeLanguage()` 和 `document.documentElement.lang`。
- 导出 `Language` 类型供组件使用。

### 切换入口

- 新组件 `src/components/layout/LanguageControl.tsx`(仿照 `ThemeModeControl` 的实现模式):Languages 图标按钮 + 菜单(中文 / English)。
- 放置位置:AdminShell 顶栏(ThemeModeControl 旁)、登录页、设置→外观页 `AppearanceCard`(与主题模式并列)。

### 文案迁移

- 26 个源文件中全部硬编码中文替换为 `useTranslation()` 的 `t("...")` 调用。
- `PageBreadcrumbs` 的 `BREADCRUMB_LABELS` 表改为存储 i18n key,渲染时翻译;`getBreadcrumbLabels` 返回 key 数组。
- `src/lib/http.ts` 的错误提示改用 `i18n.t()`(非组件上下文)。
- `main.tsx` 引入 `./i18n` 完成初始化(在渲染前)。

## 错误处理

- 缺失 key 时 i18next 回退显示 key 本身(开发时可发现);`fallbackLng: "zh"`。
- localStorage 中的非法 language 值由 zustand persist 的 merge 自然容错(仅接受 "zh" | "en",否则用检测值)。

## 测试

- 测试环境在 setup 中将 i18n 固定初始化为 `zh`,现有断言中文文案的测试保持不变。
- `PageBreadcrumbs.test.ts` 调整为断言 `getBreadcrumbLabels` 返回的 i18n key 数组(翻译发生在组件渲染层)。
- 新增 `detectLanguage()` 单元测试:zh/zh-CN/en-US/未知值各分支。
- ui store 测试补充 language 持久化与 setLanguage 行为。

## 不做的事(YAGNI)

- 不做 URL 语言参数 / 路由级语言。
- 不做日期、数字的 Intl 本地化切换。
- 不做文案命名空间拆分与懒加载。
- 不双语化 mock 业务数据。
